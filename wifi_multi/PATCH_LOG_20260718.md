## 🚀 1. 핵심 개선 사항 (Core Updates)

### ① client-logger/errorLog 전수 차단 및 로컬 격리 (적용 시점: 2026-07-17)
* **목적**: 단말 환경이 에뮬레이팅될 때 브라우저/클라이언트 내부의 미세 에러가 `/client-logger/errorLog`로 전송되어 Naver WAF 보안 필터에 탐지되는 것을 원천 차단합니다.
* **상세**:
  * 프록시 미들웨어(`addon.py`)에서 `POST /client-logger/accessLog` 이외의 에러 유도용 `POST /client-logger/errorLog` 요청을 감지하면, 서버로 전달하지 않고 **HTTP 200 (빈 Body `{}`)**을 반환해 클라이언트를 안심시킵니다.
  * 차단 사실 및 본문 에러 메시지는 로컬 세션 디렉터리에 `*_POST_client-logger_errorLog.json`으로 안전하게 저장해 두어, `monitor.sh`에서 기기 오동작 판단을 위한 Fail-Fast 감사 용도로만 활용합니다.
  * **효과**: 네이버 보안 서버에는 에러가 단 한 건도 전송되지 않아 WAF 임계치 벌점 누적을 사전에 완벽히 차단합니다.

### ② accessLog 1초 초과 지연값 비례 보정 (Proportional Capping) (적용 시점: 2026-07-17)
* **목적**: LTE 회선 병목으로 인해 웹뷰 응답 지연(`fpDuration` 또는 `networkDuration`)이 1초가 넘어갈 때, 단순 고정치로 깎을 경우 발생하는 물리적 부등식 모순(`networkDuration` > `fpDuration` 등)을 잡아내어 네이버 보안 서버의 데이터 정합성 검증 필터를 회피합니다.
* **상세**:
  * `fpDuration > 1000ms`, `networkDuration > 600ms`, `feProcessTime > 100ms` 중 하나라도 초과하는 지연 상황 발생 시 작동합니다.
  * 수집 지연값들을 난수 기반으로 비례 축소하며, 반드시 아래의 물리적 대소 관계를 만족하도록 제어합니다:
    `hashing` <= `encryption` <= `feProcessTime` <= `networkDuration` <= `fpDuration`
  * **효과**: 비상식적인 지연 패킷(1.5초~2초 대의 LTE 전송 병목)을 실제 고성능 단말 수준의 자연스러운 정합성 보장 지연값(80~800ms)으로 동적 변환하여 송신합니다.

### ③ GPS 기동 데드락 해소 및 안정적 롤백 (적용 시점: 2026-07-18 10:14)
* **목적**: 기동 시 `rptype=0` 네트워크 패킷을 확인한 후 GPS 에뮬레이터를 출발시키는 지연식 대기 로직이 모바일 클라이언트의 GPS Fix 대기 조건과 상호 충돌하여 속도가 0인 채 무한 대기하던 데드락 현상을 완전히 해소합니다.
* **상세**:
  * 가장 신뢰성이 높았던 과거 7월 2일 안정적 기동 방식(안내시작 클릭 이벤트 발생 즉시 GPS 에뮬레이터 가동)으로 원상 복귀했습니다.
  * 기동 대기 제한 버퍼 시간을 기존 30초에서 **`90초`**로 연장하여 돌발적인 무선망 및 기기 지연에 완벽하게 대응합니다.

### ④ 타임아웃 및 스케줄러 설정 전역 매개변수화 (적용 시점: 2026-07-18 10:22)
* **목적**: 기동 타임아웃 및 감시 임계치 변경을 소스 코드 수정 없이 통합 제어할 수 있도록 설정 파일에 변수화합니다.
* **상세**:

    * `USE_SUBNET_LOCK` (동일 모뎀 진입 순차 락 여부)
    * `USE_GPS_QOS_GUARD` (QoS 지연 기동 안전장치 가동 여부)
    * `STARTUP_CONNECT_TIMEOUT` (초기 curl 회선 대기 타임아웃, 기존 2초 ➡️ 10초 상향 및 3회 재시도 보강)

  * `loop.sh`, `main.sh`, `monitor.sh` 3개 엔진이 이 전역 설정을 동적으로 인가받도록 소스코드를 전체 보정하였습니다.

## 📉 3. 패치 효과 실시간 검증 (Audit Verified)

* **비례 축소 정합성 및 부등식 완벽 검증**:
  * 10시 25분대 실시간 로그 캡처 결과, 1초를 초과한 대량의 지연값이 정상 단말의 성능 수준인 600~800ms대로 비례 제어되고 있음이 확인되었습니다.
  * **예시 1**: `fp: 1139ms ➡️ 765ms` / `net: 1108ms ➡️ 137ms` / `fep: 17ms ➡️ 25ms` (부등식: `25ms(fep) <= 137ms(net) <= 765ms(fp)` 만족 ➡️ **합격**)
  * **예시 2**: `fp: 2093ms ➡️ 632ms` / `net: 390ms ➡️ 157ms` / `fep: 18ms ➡️ 20ms` (부등식: `20ms(fep) <= 157ms(net) <= 632ms(fp)` 만족 ➡️ **합격**)
* **errorLog 완벽 차단 및 로컬 격리 실적**:
  * 로그 파일 상에 `[errorLog Blocked] Intercepted and blocked errorLog POST request entirely.` 로그가 실시간 기록되고 있으며, 네이버 보안 서버로는 아예 패킷이 발송되지 않고 있음을 확인했습니다.
