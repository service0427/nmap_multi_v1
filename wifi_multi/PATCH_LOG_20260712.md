## 🚀 1. 핵심 개선 사항 (Core Updates)

### ① 10분 Cooldown 보장 및 429 임계치 100점 관대화 튜닝
* **목적**: 429 차단율이 높은 환경에서 무분별한 모뎀 물리 토글(혹사)을 방지하고 장기적인 IP 활용 효율을 극대화합니다.
* **상세**:
  * IP 할당 후 최소 **10분**은 물리적으로 IP 대역을 강제 유지하도록 수호합니다.
  * 429 차단 시 벌점 **`+1점`**, SUCCESS 완주 시 **`-2점`**으로 가감 가중치를 조절했습니다.
  * 조기 토글 발동 임계치를 기존 3점에서 **`100점`**으로 대폭 상향했습니다.
  * **효과**: 좋은 IP 대역을 만나 SUCCESS가 누적되면(하한 `-100`점) 2~3시간 동안 모뎀을 뒤집지 않고 유지하며, 완전히 불타버린 쓰레기 IP를 만났을 때만 벌점이 신속하게 누적되어 10분 쿨다운이 만료되자마자 즉시 다음 IP로 신속히 탈출(Skip)합니다.

### ② 1분 주기 헬스체크 단축 및 순차 양보 제어 (동시 토글 충돌 방지)
* **목적**: 헬스체크를 더욱 기민하게 처리하면서도 모뎀 4대의 동시 토글로 인한 USB 버스 전력 마비 및 라우팅 테이블 충돌을 원천 차단합니다.
* **상세**:
  * rotator 데몬의 루프 체크 주기 `CHECK_INTERVAL`을 300초에서 **60초(1분)**로 단축했습니다.
  * 1회 루프 내에서 물리적 토글(`smart_toggle.py` 기동)이 단 1회라도 발생하면, 동시 다발 오염 상태이더라도 나머지 모뎀은 이번 기회를 패스(`Skipping for next check.`)하고 **1분 뒤 루프에서 차례차례 안전하게 1대씩 순차 토글**하도록 양보 제어(`toggle_triggered` 플래그)를 이식했습니다.

### ③ 독점 파일 락 (Unix Exclusive File Lock) 도입
* **목적**: 주행 완료 시 트래픽 경합으로 인해 모뎀의 리셋(0점화) 상태가 옛날 스냅샷으로 역덮어쓰기되어 롤백되는 레이스 컨디션을 물리적으로 완전 방어합니다.
* **상세**:
  * `report.py`와 `lte_ip_rotator.py`가 상태 정보 파일(`lte_rotator_state.json`)을 읽고 쓸 때 Unix 수준의 파일 잠금(`fcntl.flock` 의 `LOCK_EX` / `LOCK_SH`)을 획득하도록 강제했습니다.

### ④ 일자별 히스토리 레코더 & 격리 보존
* **목적**: 일자별 점수 변동(+ / -) 기록을 투명하게 관찰할 수 있도록 이력 저장소를 확보하고, 일반 로그 세척 프로세스로부터 데이터를 보호합니다.
* **상세**:
  * `nmap-log-cleaner`의 청소 영향력을 무력화하기 위해 `logs/rotator_history/` 격리 폴더를 신설했습니다.
  * 날짜별로 자동 분할 생성되는 `scoring_history_YYYYMMDD.log` 파일에 스코어 변동 건들을 누적합니다.
  * **정교화**: 가독성을 위해 시간 포맷을 `HH:MM:SS.FFF` 로 밀리초까지 기입하며, IP 15자 정렬, 스코어 4자리 정렬을 수행해 세로줄을 칼같이 일치시켰습니다.
  * **정보 보강**: 주행의 `시작 시각 ~ 종료 시각` 구간 `(18:19:39 ~ 18:27:55)` 정보를 같이 기록하여 동일 IP 하에서의 비교 분석을 완벽하게 서포트합니다.

### ⑤ 앱 기동 캐시 보존 (⚠️ 2026-07-15 롤백/원복 완료)
* **목적**: 매 주행 기동 시마다 무자비하게 지우던 `pm clear` 명령을 제거하여 지도 앱의 자연스러운 캐시 로딩 성능을 확보하고 수집 트래픽의 자연스러움을 극대화했습니다.
* **문제점 및 조치사항 (2026-07-15)**: `rm -rf` 부분 삭제 방식으로 변경 시 `files/` 및 `cache/` 폴더 하위에 보존되는 네이버 고유 로그 트래커(Nelo SDK, GFP SDK UUID 등)가 유실되지 않아 변조된 식별자와의 불일치로 네이버 어뷰징 필터에 감지되는 문제(유효타 누락)가 확인되었습니다. 이에 완벽한 신규 단말 상태 세척을 보장하기 위해 다시 **`pm clear` 동작 방식으로 롤백 완료**했습니다.

### ⑥ 완주 증빙 SUCCESS 보고 완화
* **목적**: 실시간 네트워크 불안정 등으로 최종 `routeend` 패킷 캡처가 누락되었더라도, 목적지 도착 증빙인 플레이스 로그(`trafficjam_log.json`)가 확정 검출되었다면 SUCCESS 완료로 안전하게 마감 처리하도록 마감 연동을 복구했습니다.

### ⑦ adb-recovery 복구 모니터 예외 필터링 보강
* **목적**: 주행 기기들의 adb 연결 불안정 또는 드라이버 오동작 시 데몬이 adb reconnect를 시도하고, 예외 사항 및 재연결 안정성을 강화하여 기기 이탈 복구력을 복구했습니다.

### ⑧ API 포트 동기화
* **목적**: `config.conf` 및 `loop.sh` 의 Fallback API 포트를 최종 8013으로 완벽하게 수직 정렬 완료했습니다.
